# 미식 데이터 자산 플랫폼, 야미덱스 (YummyDex)

**1. 서비스 개요 및 브랜드 비전**

* **서비스명:** 야미덱스 (YummyDex)
* **네이밍 의미:** 'Yummy(맛있는)' + 'Index/Dex(도감/색인)'. 세상의 모든 맛을 수집하고 나만의 미식 도감을 완성해 나간다는 직관적이고 게임화(Gamification)된 의미를 담음.
* **브랜드 비전:** "과시를 위한 '광각 렌즈(인스타그램)'에서, 미식 탐구와 통제를 위한 '매크로 렌즈(야미덱스)'로." 인스타그램의 '자랑' 메커니즘을 '감각적 리스펙(Respect)'으로 치환하여, 유저의 입맛을 데이터 자산으로 만들어주는 서비스.

**2. 타겟 유저 및 페인 포인트 (Pain Points) 해결**

* **타겟 A: 미식가 (Foodies)**
* **기존 문제:** 인스타그램의 사진 보정 압박, 광고성 맛집 정보의 범람, 분위기 위주의 획일화된 리뷰.
* **야미덱스의 해결:** 테이블 전체가 아닌 가장 감동한 '딱 한입'에만 집중. 주관적 감각을 정밀하게 기록하고, 타인과 입맛 싱크로율을 바탕으로 진짜 정보를 교류.


* **타겟 B: 다이어터 (Dieters)**
* **기존 문제:** 기존 식단 앱의 빡빡한 칼로리/탄단지 수치 입력 강박, 식단을 어겼을 때 느끼는 죄책감과 기록 포기.
* **야미덱스의 해결:** 전체 식사가 아닌 내가 먹은 '특정 부위'만 기록하는 부분 통제 허용. "400kcal 초과"라는 경고 대신, "단짠 밸런스 80%", "치팅 지수 상위 10%" 등의 유머러스한 지수(Index)로 표현하여 기록의 스트레스를 낮춤.



**3. 인스타그램과의 핵심 차별점 (패러다임 전환)**

* **기록의 문법: [전시] vs [마킹]**
* 인스타그램: 분위기, 인테리어, 예쁜 플레이팅 등 '상황'을 전시함.
* 야미덱스: 수많은 음식 중 내 혀를 사로잡은 '특정 부위'에 동그라미(점)를 치는 '마킹'에 집중함.


* **검색의 질: [광고/무드] vs [논리/맛]**
* 인스타그램: 해시태그 위주의 광고성 피드 나열.
* 야미덱스: 유저가 남긴 맛 태그(단/짠/바삭/눅눅 등)를 기반으로 필터링 된 논리적 미식 큐레이션 제공.


* **소셜 상호작용: [부러움] vs [기여와 존중]**
* 인스타그램: 타인의 삶을 훔쳐보며 느끼는 부러움과 박탈감.
* 야미덱스: 부족한 정보를 서로 채워주는 집단지성. "내 감각이 타인에게 인정받고 유용한 데이터가 된다"는 효능감.



**4. 핵심 UX/UI 로직: 빠르고 쾌적한 기록**

* **1단계 - 찍고 마킹하기 (1초):** 앱을 켜고 사진을 찍은 뒤, 화면에서 가장 인상 깊었던 부위에 터치하여 동그라미/점을 남김.
* **2단계 - 짧게 맛 뱉기 (3초):** 구구절절한 텍스트 리뷰 대신, 객관식 슬라이더나 직관적인 맛 태그(단짠, 맵느, 자극적 등)를 선택하여 감정을 기록함. 업로드 과정은 여기서 즉시 종료되어 '노동'의 느낌을 지움.
* **3단계 - 비동기적 정보 보충 (알림 유도):** * 시스템이 작성자를 닦달하지 않음.
* 일정 시간 후, "당신과 입맛이 비슷한 5명이 이 야미덱스에 관심을 보이고 있어요! 식당 위치를 알려주실래요?"라는 푸시 알림을 통해 '인기인의 특권'처럼 기분 좋게 상세 정보 입력을 유도함.



**5. 소셜 및 커뮤니티 정책: '평가'가 아닌 '취향 매칭'**

* **미식 탐정 (집단지성 시스템):** 식당 이름이나 메뉴를 모를 때 '탐정 요청'으로 게시. 다이어터의 경우 "이 소스 다이어트 중에 먹어도 되나요?"라고 올리면, 커뮤니티 고수들이 성분 정보나 팁을 댓글로 채워주며 정보가 완성됨.
* **피드백 시스템의 혁신 (서열화 방지):** * '점수(Score)' 시스템 폐지: 미식에 점수를 매겨 유저 간 서열을 나누거나 박탈감을 주는 것을 방지함.
* 유니크한 리액션 버튼: 단순한 '좋아요/싫어요' 대신, **[리스펙(Respect) / 나랑 똑같아요 / 오 신기해요 / 나랑은 달라요]** 등의 취향 중심 버튼 운영.


* **입맛 필터링 (악플 방어):** 공격적인 댓글("이게 왜 맛있음?")이 달리더라도, 시스템이 "당신과 입맛이 80% 다른 유저의 의견입니다"라고 매칭도를 함께 보여주어, '틀린 것'이 아니라 '취향이 다를 뿐'임을 명확히 인지시킴.

**6. 비즈니스 확장성 (Killer Loop)**

* **미식 MBTI (성적표):** 유저의 누적된 마킹 데이터를 분석하여 "당신은 10%의 독특한 취향을 가진 개성파", "당신은 염도 7 이상을 즐기는 자극 러버" 등의 개인화된 분석 리포트 제공.
* **레시피/내식(內食) 연동:** 외식이나 배달 음식의 특정 소스, 고명 등에 동그라미를 치고 열광하면, AI가 '만개의레시피' 등과 연동하여 **가장 유사한 맛을 집에서 구현할 수 있는 레시피**를 추천. 외식(소비)의 경험을 내식(생산)으로 강제 연결하는 독보적 BM 구축.
