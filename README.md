# 미식 데이터 자산 플랫폼, 야미덱스 (YummyDex)

**1. 서비스 개요 및 브랜드 비전**

* **서비스명:** 야미덱스 (YummyDex)
* **네이밍 의미:** 'Yummy(맛있는)' + 'Index/Dex(도감/색인)'. 세상의 모든 맛을 수집하고 나만의 미식 도감을 완성해 나간다는 직관적이고 게임화(Gamification)된 의미를 담음.
* **브랜드 비전:** "과시를 위한 '광각 렌즈(인스타그램)'에서, 미식 탐구와 데이터 기록을 위한 '매크로 렌즈(야미덱스)'로." 인스타그램의 '자랑' 메커니즘을 '감각적 리스펙(Respect)'으로 치환하여, 유저의 입맛을 정밀한 데이터 자산으로 만들어주는 서비스.

**2. 타겟 유저 및 페인 포인트 (Pain Points) 해결**

* **타겟 A: 미식가 (Foodies)**
* **기존 문제:** 인스타그램의 사진 보정 압박, 광고성 맛집 정보의 범람, 분위기 위주의 획일화된 리뷰.
* **야미덱스의 해결:** 전체적인 분위기나 인테리어가 아닌, 내가 느낀 **'음식 본연의 맛'**에만 집중. 주관적 감각을 정교한 맛 태그로 기록하고, 타인과의 입맛 싱크로율을 바탕으로 신뢰할 수 있는 미식 정보를 교류.

* **타겟 B: 다이어터 (Dieters)**
* **기존 문제:** 기존 식단 앱의 빡빡한 칼로리/탄단지 수치 입력 강박, 식단을 어겼을 때 느끼는 죄책감과 기록 포기.
* **야미덱스의 해결:** 전체 식사량이 아닌 내가 선택한 **'맛의 정수'**를 기록하여 심리적 부담 완화. "400kcal 초과"라는 경고 대신, "단짠 밸런스 80%", "치팅 지수 상위 10%" 등의 유머러스한 지수(Index)로 표현하여 기록의 지속성을 높임.

**3. 인스타그램과의 핵심 차별점 (패러다임 전환)**

* **기록의 문법: [전시] vs [데이터화]**
* 인스타그램: 분위기, 인테리어, 예쁜 플레이팅 등 '상황'을 전시함.
* 야미덱스: 수많은 음식 중 내 혀를 사로잡은 **'맛의 포인트'**를 정밀하게 태깅하고 데이터로 축적함.

* **검색의 질: [광고/무드] vs [논리/맛]**
* 인스타그램: 해시태그 위주의 광고성 피드 나열.
* 야미덱스: 유저가 남긴 미세한 맛 태그(단/짠/바삭/눅눅 등)를 기반으로 필터링 된 논리적 미식 큐레이션 제공.

* **소셜 상호작용: [부러움] vs [기여와 존중]**
* 인스타그램: 타인의 삶을 훔쳐보며 느끼는 부러움과 박탈감.
* 야미덱스: 부족한 정보를 서로 채워주는 집단지성. "내 감각 데이터가 타인에게 인정받고 유용한 정보가 된다"는 효능감 제공.

**4. 핵심 UX/UI 로직: 빠르고 쾌적한 기록**

* **1단계 - 찍고 감각 포착하기 (1초):** 앱을 켜고 음식 사진을 촬영하거나 선택함. 화려한 보정 없이 맛의 질감이 잘 드러나는 사진이면 충분함.
* **2단계 - 짧게 맛 뱉기 (3초):** 구구절절한 텍스트 리뷰 대신, 직관적인 맛 태그(단짠, 맵느, 고소, 바삭 등)와 슬라이더를 통해 내 혀가 느낀 감각을 즉시 기록함. 업로드 과정이 매우 단순하여 기록의 피로도를 최소화함.
* **3단계 - 비동기적 정보 보충 (알림 유도):** 시스템이 작성자를 닦달하지 않음. 일정 시간 후, "당신과 입맛이 비슷한 5명이 이 미식 데이터에 관심을 보이고 있어요! 상세 위치를 공유해주실래요?"라는 알림을 통해 기분 좋게 상세 정보를 보충함.

**5. 소셜 및 커뮤니티 정책: '평가'가 아닌 '취향 매칭'**

* **미식 탐정 (집단지성 시스템):** 식당 이름이나 메뉴를 모를 때 '탐정 요청'으로 게시. 커뮤니티 고수들이 사진과 맛 묘사를 보고 정보를 채워주며 데이터가 완성됨.
* **피드백 시스템의 혁신 (서열화 방지):** 
  * '점수(Score)' 시스템 폐지: 미식에 일률적인 점수를 매겨 유저 간 서열을 나누는 행위를 배제함.
  * 유니크한 리액션 버튼: 단순한 '좋아요' 대신, **[리스펙(Respect) / 나랑 똑같아요 / 오 신기해요 / 나랑은 달라요]** 등의 취향 중심 버튼 운영.
* **입맛 필터링 (취향 존중):** "이게 왜 맛있음?" 식의 공격적인 반응에 대해, 시스템이 "당신과 입맛이 80% 다른 유저의 의견입니다"라고 매칭도를 함께 보여주어 '다름'을 명확히 인지시킴.

**6. 비즈니스 확장성 (Killer Loop)**

* **미식 MBTI (성적표):** 누적된 맛 데이터를 분석하여 "염도 7 이상을 즐기는 자극 러버", "바삭한 식감에 집착하는 텍스처 매니아" 등 개인화된 분석 리포트 제공.
* **맞춤형 큐레이션:** 외식에서 열광했던 특정 맛(예: 특정 소스의 감칠맛 등)을 분석하여, 이와 유사한 맛을 구현할 수 있는 밀키트나 레시피를 추천하는 BM 구축.
