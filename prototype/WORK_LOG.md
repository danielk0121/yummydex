## 1. 작업 개요

README.md 기획서를 기반으로, 야미덱스(YummyDex)의 핵심 UX/UI를 모바일 앱 스타일의 싱글 페이지 프로토타입 HTML로 구현하였습니다.

**대상 파일:** `prototype/index.html`

## 2. 구현된 화면 (총 6개)

### 2-1. 홈 피드
- 입맛 싱크로율 기반 피드 정렬 (입맛 매칭 / 최신 / 인기 탭)
- 음식 사진 위 **마킹 도트** 애니메이션 (가장 감동한 부위 표시)
- **맛 태그** 표시 (바삭, 짭조름, 고소, 달콤, 매움 등)
- **취향 리액션 버튼** 4종: 리스펙 / 나랑 똑같아요 / 오 신기해요 / 나랑은 달라요
- 입맛 싱크로율 바 인디케이터
- 악플 방어 UX: "입맛이 58% 다른 유저의 의견입니다" 표시

### 2-2. 탐색
- 맛/식감/재료 검색바
- **맛 태그 필터 칩** 가로 스크롤 (단맛, 짠맛, 매운맛, 바삭, 고소, 부드러움, 쫄깃)
- 2열 그리드 큐레이션 카드 (마킹 도트 + 맛 태그 포함)
- 기획서의 "논리적 미식 큐레이션" 컨셉 반영

### 2-3. 마킹 (중앙 촬영 버튼)
기획서의 **핵심 UX 3단계 플로우**를 인터랙티브하게 구현:

| 단계 | 소요 시간 | 기능 |
|------|-----------|------|
| ① 찍기 | 1초 | 카메라 UI → 셔터 버튼 탭 |
| ② 마킹 | 1초 | 사진 위 감동 부위 터치 → 동그라미 마킹 생성 |
| ③ 맛 뱉기 | 3초 | 맛 태그 8종 객관식 선택 + 감동 강도 슬라이더 |

- 단계별 상단 진행 표시 (현재 단계 하이라이트)
- 마킹 완료 시 토스트 알림 후 홈으로 자동 이동

### 2-4. 미식 탐정
- 집단지성 시스템 배너 + "탐정 요청하기" CTA
- 탐정 요청 카드 리스트 (수사 중 / 해결됨 배지)
- 답변 수, 조회수, 유력 용의자 정보 표시
- **다이어터 질문 지원** 예시: "이 소스 다이어트 중 먹어도 될까요?"

### 2-5. 내 도감 (프로필)
- 프로필 헤더 (아바타, 닉네임, 마킹/리스펙/탐정해결 통계)
- **미식 MBTI 카드**: "S·C·B·I 자극 러버 × 크리에이티브 미식가"
  - 상위 10% 독특한 취향, 염도 선호도, 단짠 밸런스, 치팅 지수 등
- **맛 프로파일 바 차트**: 단맛/짠맛/매운맛/바삭/고소/부드러움 6축 시각화
- 최근 마킹 3열 그리드 갤러리

### 2-6. 알림
기획서의 **비동기적 정보 보충 유도** UX 구현:
- 🤝 입맛 매칭 알림: "입맛이 87% 일치! 5명이 관심을 보이고 있어요!" → "위치 알려주기" CTA
- 🔄 리액션 알림: "나랑 똑같아요 리액션을 남겼어요"
- 🔍 탐정 업데이트: 새 답변 알림
- 📊 주간 미식 리포트: "이번 주 '바삭' 태그를 가장 많이 선택했어요"
- 💡 레시피 추천 연동: "바삭한 맛을 집에서 재현해볼까요?"

## 4. 기술 구현 사항

- **프레임워크:** 순수 HTML/CSS/JS (의존성 없음)
- **레이아웃:** 모바일 퍼스트, max-width 430px 앱 프레임
- **화면 전환:** CSS transform + opacity 트랜지션 기반 SPA
- **인터랙션:** 터치 마킹, 태그 토글, 슬라이더, 토스트 알림
- **애니메이션:** 마킹 도트 pulse, 마크 생성 scale, 패널 slide-up, 토스트 fade-in
- **하단 네비게이션:** 5탭 (홈, 탐색, 마킹 중앙버튼, 탐정, 내 도감)
