# 미식 리액션 버튼 분석 및 개선 제안

## 1. 현황 분석
현재 야미덱스(YummyDex)의 피드에는 다음과 같은 4가지 리액션 버튼이 존재합니다.
- **리스펙 (Respect)**: 작성자의 미식 감각에 대한 존중
- **나랑 똑같아요**: 동일한 미각 경험 공유
- **오 신기해요**: 새로운 맛 표현에 대한 호기심
- **나랑은 달라요**: 취향의 차이 인정

## 2. 문제점 (논리적 존재 이유의 결여)
- **모호한 데이터 가치**: '리스펙'이나 '신기해요'는 감성적인 영역으로, 야미덱스가 지향하는 '미각의 디지털화' 및 '정밀한 미식 지표' 구축에 기여하는 바가 낮음.
- **데이터 중복**: 야미덱스는 이미 '입맛 싱크로율(%)'이라는 강력한 데이터 비교 우위를 가지고 있음. 감성적인 버튼은 이 수치 기반의 신뢰도를 희석시킬 우려가 있음.
- **기능적 실효성 부족**: 단순 '좋아요'의 변주일 뿐, 유저가 피드백을 통해 얻는 실질적인 정보(예: "이 사람의 정보는 나에게 유용한가?")를 직접적으로 나타내지 못함.

## 3. 수정 제안 방향
감성적 리액션을 배제하고, **데이터의 신뢰도와 정보의 유용성**을 강화하는 방향으로 개편을 제안합니다.

### [안 A] 신뢰도 및 유용성 중심 (추천)
- **도움돼요 (Helpful)**: 이 미식 데이터가 나의 다음 식사 결정에 실질적인 도움이 되었는가? (데이터의 유용성 검증)
- **나랑 똑같아요 (Same Taste)**: 나의 실제 미각 경험과 데이터가 일치하는가? (데이터의 정확성 검증)
- **가보고 싶어요 (Wishlist)**: 이 미식 지표를 보고 직접 경험해보고 싶은 욕구가 생겼는가? (행동 유도)

### [안 B] 미식 지표 피드백 중심
- **정확해요 (Accurate)**: 제시된 20대 미식 지표 수치가 실제 맛을 잘 대변하고 있는가?
- **공감해요 (Sympathy)**: 주관적 맛 묘사에 동의하는가?

## 4. 최종 결정 및 코드 반영 계획
사용자님의 의견에 따라 **[안 A]**를 기반으로 버튼을 단순화하고 의미를 명확히 하겠습니다.
- **기존**: 리스펙, 나랑 똑같아요, 오 신기해요, 나랑은 달라요
- **변경**: **도움돼요 👍 / 가보고 싶어요 📍 / 나랑 똑같아요 🔄** (3종으로 축약)
  - '나랑은 달라요'는 '입맛 싱크로율' 시스템이 이미 수행하고 있으므로 버튼에서 제외.
