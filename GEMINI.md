# LLM 규칙
*LLM 은 이 규칙을 숙지하고 반드시 따라야 한다!*

> **[현재 적용 규칙]**
> 지금은 **A규칙**을 적용한다. 다른 규칙은 적용하지 않는다.

## [A규칙] 표준 작업 및 메인 머지 워크플로우
- 1개 작업 단위로 코드를 수정하고, 커밋한다.
  - 1개 작업을 시작할 때 main 브랜치에서 새로운 브랜치를 만들어서 작업한다.
  - main 에서 새로운 브랜치를 만들기 전에 fetch 하여 origin 에서 내용을 받아 온 후 진행한다.
- **작업 완료 및 머지**:
  - 작업을 마치면 해당 브랜치를 origin에 push한다.
  - push 후에는 main 브랜치로 전환하여 작업한 브랜치를 머지(merge)한다.
  - **머지 완료 후 로컬 main 브랜치를 origin main에 반드시 push한다.**
- **브랜치 보존**: 작업 완료 후 main에 머지하더라도, 개별 작업 브랜치는 삭제하지 않고 보존한다.

## [B규칙] 독립 브랜치 작업 워크플로우 (메인 머지 안 함)
- 1개 작업 단위로 코드를 수정하고, 커밋한다.
  - 1개 작업을 시작할 때 main 브랜치에서 새로운 브랜치를 만들어서 작업한다.
  - main 에서 새로운 브랜치를 만들기 전에 fetch 하여 origin 에서 내용을 받아 온 후 진행한다.
- **작업 완료**:
  - 작업을 마치면 해당 브랜치를 origin에 push하고 작업을 종료한다.
  - **main 브랜치로 전환하거나 머지(merge)하지 않는다.**
- **브랜치 보존**: 생성된 모든 작업 브랜치는 삭제하지 않고 보존한다.

## 점진적 반영 규칙 (Beta to Prototype)
- 모든 새로운 기능 개발이나 대규모 수정은 1차적으로 **beta** 폴더 하위의 파일에서 먼저 진행한다.
- beta 폴더에서의 작업 결과가 만족스럽고 안정적이라고 판단되면, 해당 내용을 **prototype** 폴더로 복사/이동하여 반영한다.
- prototype 폴더는 항상 검증된 최신 상태를 유지하도록 관리한다.

## 공통 규칙
- **prototype 및 beta 하위의 파일을 수정 할 때에만 이 내용을 적용한다.**
  - 최상단에 현재 시간값의 버전 정보를 수정 후 커밋한다.
  - 대상 파일 : prototype/*.html, beta/*.html 파일들
  - 버전 정보 형식 : 버전 정보(`v-yyMMdd-HHmm-kst`)를 현재 시간값으로 수정
  - 예: 현재시간이 2026-03-15 16:30 KST 이면 v-260315-1630-kst 를 사용한다.
- 모든 문서와 커밋 메세지, 코드 주석 등 한글을 사용한다.
- LLM 응답 또한 한글을 사용한다.
