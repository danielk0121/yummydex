# LLM 규칙
*LLM 은 이 규칙을 숙지하고 반드시 따라야 한다!*

## 작업 규칙
- **1개 작업 단위**로 코드 수정 후 커밋
- 작업 시작 시 `main` 브랜치에서 새 브랜치 생성 (먼저 `fetch`)
- 작업 완료 후: 브랜치를 origin에 push
- 작업 브랜치는 삭제하지 않고 보존

## 버전 정보 규칙
- `prototype/*.html` 파일 수정 시에만 적용
- 최상단에 버전 정보 수정: `v-yyMMdd-HHmm-kst` 형식
- 예: 2026-03-12 00:56 KST → `v-260312-0056-kst`

## 언어 규칙
- 모든 문서, 커밋 메시지, 코드 주석, LLM 응답 → **한글 사용**
